# CRM ↔ Teroxx Portfolio App — field mapping

This document specifies how a CRM (HubSpot, Salesforce, or any other)
should map onto the Teroxx Portfolio App's `Client` and `ClientLot`
records, and how external systems should authenticate against the v1
API. Phase 9 ships the seam only; the actual HubSpot / Salesforce
clients live behind whichever integration we build next.

---

## Authentication

All `/api/v1/*` endpoints require a bearer token in the
`Authorization: Bearer tax_<prefix>_<secret>` header. Tokens are minted
by a logged-in advisor via:

```
POST /api/admin/api-tokens
Content-Type: application/json

{
  "name": "HubSpot Production",
  "provider": "hubspot",
  "scopes": "clients:read,clients:write"
}
```

The response includes a `plaintext_token` field which is shown **once**;
store it in the CRM's secret manager immediately. The server only
retains a salted SHA-256 hash plus the 16-character prefix for lookup.

Tokens can be revoked at any time via
`DELETE /api/admin/api-tokens/{id}`; revocation is immediate.

**Scopes**

| Scope             | Allows                                              |
|-------------------|-----------------------------------------------------|
| `clients:read`    | `GET /api/v1/clients` and `GET /api/v1/clients/{id}` |
| `clients:write`   | `POST` / `PATCH` on the above                       |
| `webhooks:receive`| `POST /api/v1/webhooks/{provider}` (reserved)       |
| `admin`           | Implies all of the above                            |

## Endpoints

```
GET    /api/v1/clients                    → list active clients
GET    /api/v1/clients/{client_id}        → read one client (with lots)
POST   /api/v1/clients                    → create a client
PATCH  /api/v1/clients/{client_id}        → update top-level fields
POST   /api/v1/webhooks/{provider}        → receive a CRM webhook
```

All payloads are JSON. Timestamps in responses are ISO-8601 in UTC.
Money fields are floats in the client's reporting currency, except
`starting_capital_usd` which is always USD.

## Field mapping — `Client`

| Teroxx field             | Type      | HubSpot Contact field          | Salesforce Account field        | Notes |
|--------------------------|-----------|--------------------------------|---------------------------------|-------|
| `id`                     | string    | `external_id` custom property  | `External_Id__c`                | Generated by Teroxx on create; CRM should store and round-trip it |
| `name`                   | string    | `firstname` + `lastname`       | `Name`                          | Free-text; required on create |
| `profile`                | enum      | `risk_profile` custom property | `Risk_Profile__c`               | One of: `Conservative`, `Balanced`, `Growth`, `Aggressive` |
| `domicile`               | string    | `country`                      | `BillingCountry`                | Free-text city + country, e.g. "Munich, DE" |
| `domicile_country`       | ISO-2     | `country_code` custom property | `BillingCountryCode`            | Drives MiCA disclaimer selection |
| `currency`               | ISO-3     | `reporting_currency` custom    | `CurrencyIsoCode`               | One of `USD`, `EUR`, `GBP`, `CHF`, `AED` |
| `inception_date`         | YYYY-MM-DD| `inception_date` custom        | `Inception_Date__c`             | Day the advisory mandate started |
| `starting_capital_usd`   | float     | `starting_capital_usd` custom  | `Starting_Capital_USD__c`       | Reported in USD regardless of `currency` |
| `tagline`                | string    | `client_tagline` custom        | `Description`                   | One-line framing; rendered on cover page |
| `risk_notes`             | string    | `risk_notes` custom            | `Risk_Notes__c`                 | Free-form internal note |
| `implementation_note`    | string    | `implementation_note` custom   | `Implementation_Note__c`        | Appears on the proposal PDF implementation page |
| `created_at` / `updated_at` | ISO-8601 | n/a                           | n/a                             | Read-only |

## Field mapping — `ClientLot`

A lot represents a single tax-lot purchase. Multiple lots in the same
ticker are kept separately so qty-weighted entries can be computed.

| Teroxx field   | Type    | HubSpot Deal field          | Salesforce Asset field        |
|----------------|---------|-----------------------------|-------------------------------|
| `id`           | int     | `external_lot_id`           | `External_Lot_Id__c`          |
| `client_id`    | string  | `associated_contact_id`     | `AccountId`                   |
| `ticker`       | string  | `asset_ticker`              | `Asset_Ticker__c`             |
| `quantity`     | float   | `quantity`                  | `Quantity`                    |
| `entry_price`  | float   | `entry_price_usd`           | `Entry_Price_USD__c`          |
| `entry_date`   | date    | `closedate`                 | `Entry_Date__c`               |
| `notes`        | string  | `description`               | `Description`                 |

## Webhooks

Webhooks are accepted at `POST /api/v1/webhooks/{provider}` where
`provider` is one of `hubspot`, `salesforce`, or any other tag of your
choice. The request must include an `X-Teroxx-Signature` header
containing `sha256=<hex>` of the raw body using the per-provider shared
secret configured server-side in `WEBHOOK_SECRETS` (a Python dict in
`app/api_v1.py`).

Request body shape (recommended):

```json
{
  "event": "contact.updated",
  "external_id": "12345",
  "fields": { "risk_profile": "Balanced" }
}
```

The Phase 9 receiver records every valid webhook to `advisor_actions`
with `action_type = "webhook_received"`. Downstream behaviour (auto-
update clients, push notifications, etc.) is left to the integration
that will be built later.

## Operational notes

- The CRM is the system of record for client metadata (name,
  jurisdiction, suitability). The portfolio app is the system of record
  for allocation, lots, P&L and the audit trail.
- Soft-deletes on the Teroxx side set `deleted_at` and hide the client
  from `GET /api/v1/clients`. The CRM should treat a 404 on a known
  external ID as "the client was removed in Teroxx" rather than as an
  integration error.
- All v1 endpoints log to `advisor_actions` with `actor_email =
  "token:<token_name>"` so compliance reviews can attribute external
  changes to a specific integration token.
